Boost - The Boost C++ libraries are a set of free software libraries that extend the functionality of C++.

    Используется в исходном коде РДО.

0. Проверяем, что на компьютере есть не меньше 1.8Гб свободного места.

1. Скачиваем boost
    http://www.boost.org/
    проверялось на версии 1.55

    Для Windows предположим, что разархивировали в 'с:\boost_1_55_0_src'
    Для GNU/Linux предположим, что разархивировали в '/home/user/boost'

2. Собираем статические либы буста
2.1. Для Windows

    cd c:\boost_1_55_0_src
    bootstrap.bat

    Если вывелось
    Bootstrapping is done. To build, run:

        .\b2
    , то всё хорошо, переходим к 2.3.

    Иначе смотрим на ошибку. У меня в путях (переменная PATH) было с виду правильное имя
    "C:\Program Files\Microsoft DirectX SDK (August 2006)\Utilities\Bin\x86"
    Оно в кавычках, и  инсталятор упорно ругался на
    \Utilities\Bin\x86";"C:\Program...
    Заходим в редактор переменных окружения и удаляем все двойные кавычки из PATH. Делаем это в двух местах: для пользователя и общесистемные.
    Повторяем 2.1.

2.2. Для GNU/Linux:

    cd /home/user/boost
    ./bootstrap.sh

2.3. Для Windows

2.3.1 Visual Studio (дочитай до конца этой главы)

    Создать папку, в которую будет проинсталлирован буст, например, C:\boost_1_55_0

    b2 debug release link=static threading=multi -j n install --prefix=C:\boost_1_55_0
    , где n - количество процессоров (точнее, одновременно собираемых исходников)

    для i7 пишем
    b2 debug release link=static threading=multi -j 8 install --prefix=C:\boost_1_55_0

    для i5 пишем
    b2 debug release link=static threading=multi -j 4 install --prefix=C:\boost_1_55_0

    Задать переменную окружения
    BOOST_ROOT=C:\boost_1_55_0

2.3.2 MinGW

    Поставить MASM32 SDK (http://www.masm32.com/masmdl.htm) и прописать путь
    set PATH=C:\masm32\bin;%PATH% (это можно сделать только в консоле сборки, а после сборки буста удалить MASM32 с винта)

    cd c:\boost_1_55_0_src
    bootstrap.bat mingw
    b2 debug release link=static threading=multi -j 4 install --prefix=C:\boost_1_55_0 toolset=gcc

    Если вы уже скомпилили буст под Visual Studio, то ничего страшного.
    Библиотеки буста после компиляции MinGW и Visual Studio имеют разные имена и располагаются в одном месте.

2.4 Для GNU/Linux запускаем b2 с параметрами

    ./b2 --layout=tagged --build-type=complete --prefix=/home/user/boost-install threading=multi -j 8 install

    export BOOST_ROOT=~/boost-install

3. Если потребуется пересобрать буст, то

3.1 b2 --clean-all
3.2 удаляем папку lib из C:\boost_1_55_0\bin.v2
3.3 удаляем папку lib из C:\boost_1_55_0\stage

    и повторяем шаг 2

    PS: Уже не в первый раз понадобилось описание имён бибилиотек, вот оно
    http://www.boost.org/doc/libs/1_55_0/more/getting_started/unix-variants.html#library-naming

    PSS: Параметры запуска b2
    http://www.boost.org/boost-build2/doc/html/bbv2/overview/invocation.html#bbv2.overview.invocation.properties
