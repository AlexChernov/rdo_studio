Инструкция установки РДО из исходников

1. Работа над ошибками (актуально для винды)
Обязательно проверь правильность установки install.svn.txt, на предмет "command line client tools":
Открой консоль (cmd.exe), запуски svn. Если будет
Type 'svn help' for usage.
, то всё хорошо. Иначе, просто переустанови черепаху с нужной галкой.

2. Больше не хочется напоминать (актуально для винды)
2.1. В инструкции будет куча команд с консоли. По умолчанию будем считать, что нам нужна правильно настроенная на компиляцию консоль. В линухе так и есть, а под виндой надо запустить:

- если установлена Visual Studio 2008
Пуск\Microsoft Visual Studio 2008\Visual Studio Tools\Visual Studio 2008 Command Prompt

- если установлена Visual Studio 2012
Пуск\Microsoft Visual Studio 2012\Visual Studio Tools\Developer Command Prompt for VS2012

- если установлена Visual Studio 2013
Пуск\Microsoft Visual Studio 2013\Visual Studio Tools\Developer Command Prompt for VS2013

2.2. Может понадобится батник vcvars32.bat
Ищи его в папке VC/bin вижуальника, например
"C:\Program Files (x86)\Microsoft Visual Studio 9.0\VC\bin\vcvars32.bat"
"C:\Program Files\Microsoft Visual Studio 12.0\VC\bin\vcvars32.bat"

Если его надо вызвать его из консоли, то кавычки обязательны

2.3. В новой версии Visual Studio (наткнулся в 2013) удалены настройки сторонних библиотек. Ранее пути к ним прописывались в настройках

Tools/Options/Projects and Solutions/VC++ Directories
Теперь в другом месте. Описание решения http://msdn.microsoft.com/en-us/library/vstudio/ee855621%28v=vs.120%29.aspx (взято с этой же страницы настроек, жмаканьем по '?')

Было довольно удобно: читаем инструкцию к boost и прописываем значения. Теперь это сделать невозможно, т.к. нужен солюшен rdo, которого еще просто нет. Чтобы не забыть "прописать пути": берём топор, идём в лес и делаем зарубку на берёзе. И так минимум два раза: для boost и Qt
