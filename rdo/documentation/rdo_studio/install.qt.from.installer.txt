0. Найти на винте до 1.7Gb
 + установленная MSVC

1. Загрузка Qt SDK
http://qt-project.org/downloads

Находим и загружаем последнюю стабильную версию Qt в зависимости от платформы, разрядности и вашего компилятора

Примечания: для загрузки нужен хороший интернет и в качестве браузера лучше использовать opera с докачной, в chrome закачка часто обрывается на 80-90%

2. Установка

В окне выбора компонентов лучше выбрать всё
Мы используем Qt по лицензии LGPL

2.1 Под виндой
Просто запустить только что скаченный инсталлятор

2.2 Под линухами (посмотри на 2.2.1)
Установить в директория по умолчанию
/home/<username>/Qt5.*.*
с правами текущего пользователя или в директорию
/opt/qt/5.*.*
от root или из под sudo

2.2.1 Под Ubuntu 13.10 сделал, как написано, но стал получать сообщение
could not find a Qt installation of ''
при вызове консольных qt-приложений, например,
qmake
qcollectiongenerator

Видимо, в Ubuntu уже стояла какая-то версия Qt и начался конфликт
Решение:
- снести установленную из инсталятора Qt
- доставить штатную Qt через пакеты
sudo apt-get install qt5-default qttools5-dev qttools5-dev-tools

2.2.2. Под Debian Wheezy еще необходима библиотека GL. Пакет libgl1-mesa-dev содержит достаточную реализацию OpenGL API.
sudo apt-get install libgl1-mesa-dev

3. После установки необходимо определить две переменные окружения: QTDIR и PATH

3.1. Windows

Находим директорию, куда поставили Qt, в её содержимом ищем библиотеку. Её корень должен выглядеть так
bin\
doc\
examples\
imports\
include\
lib\
и т.д.

И прописываем QTDIR, например
QTDIR=D:\Qt\Qt5.0.1\5.0.1\msvc2010
PATH = %QTDIR%/bin

3.2. Linux
QTDIR=/home/<username>/Qt5.*.*/5.*.*/(gcc/gcc64)
или
QTDIR=/opt/qt/5.*.*/5.*.*/(gcc/gcc64)

PATH=$PATH:$QTDIR/bin
